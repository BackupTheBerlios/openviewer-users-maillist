> I think it's expecting OpenViewer.ini to exist in bin as well (only
> OpenViewer.ini.example is included in SVN, though).  In the latest
> revision, mkapp.sh should correctly determine whether OpenViewer.ini
> exists or not, and include it if it is found.
>
> If you want to change any of the settings it uses when it starts up
> (UI, rendering engine, loginuri, etc.), then you should copy
> OpenViewer.ini.example to OpenViewer.ini, and make changes in
> OpenViewer.ini before running mkapp.sh.  (You can also modify
> OpenViewer.app/Contents/Resources/OpenViewer.ini after the .app has
> been built)
